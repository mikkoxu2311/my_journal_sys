# my_journal_sys
A journal system templates works with Claude Code skill

## 🎯 系统概览

这是一个三层journal系统，帮助你：
- ✅ 捕获感悟、想法、素材（不再遗忘）
- ✅ 定期反思和校准（保持方向）
- ✅ 将想法转化为行动（真正改变）

```
Layer 1: 快速捕获层 → Layer 2: 提炼加工层 → Layer 3: 应用层
   (每日记录)           (每周处理)            (持续行动)
```

---

## 📁 模板清单

### 日常使用（Daily）

1. **Daily Journal.md** - 标准版每日journal
   - 包含早晨设定、三层挖掘、快速标记、晚间反思，如果每天能抽出10-15分钟记录，状态正常的日子

2. **Daily Journal - Low Energy.md** - 简化版
   - 只记录基本状态和能量流向，适合低能量、疲惫的日子

3. **Daily Journal - Reflection.md** - 对话式
   - 如果有纠结/困惑需要思考，通过自我对话理清困惑

4. **Daily Journal - Project Day.md** - 项目复盘
   - 记录项目推进和学习，完成了milestone或推进了重要项目时使用

5. **Daily Journal - Parenting Day.md** - 带娃日
   - 记录和孩子的瞬间，主要捕获内容素材

6. **Day Interrupt Check-in.md** - 日间打断
   - 设置随机提醒，打破自动驾驶，每天3次，每次1分钟

### 定期回顾（Review）- 自动生成

7. **Weekly Review.md** - 每周进行，用calude skills
   - 从journal提取insights/ideas/seeds
   - 分析能量流向和行为模式
   - 选择下周要推进的想法

8. **Monthly Reflection.md** - （待更新）
   - 反向校准（anti-vision）
   - 挖掘隐藏目标和行为模式
   - 身份检查和调整

9. **Quarterly Deep Dive.md** - （待更新）
    - 深度心理挖掘（4-6小时）
    - 设计anti-vision和vision
    - 制定季度系统（video game framework）

### 持续管理（Pools）- 自动生成

10. **Insights Pool.md** - 感悟池
    - 集中存放所有insights
    - 每月review，提炼成原则

11. **Ideas Backlog.md** - 想法待办
    - 管理所有想法，按优先级
    - 记录放弃原因，避免重复纠结

12. **Content Seeds.md** - 内容种子
    - 存放可能成为内容的素材
    - 每周选1-2个发展

## 💡 使用方法

### 1. 选择合适的模板

**如何选择Daily Journal模板？**
- 状态好，有10-15min时间 → Daily Journal（标准版）
- 状态差，很累 → Daily Journal - Low Energy
- 有困惑纠结 → Daily Journal - Reflection
- 完成了项目 → Daily Journal - Project Day
- 主要在带娃 → Daily Journal - Parenting Day

### 2. 标记系统

**什么时候标记？**
- 写journal时随手标记，不要事后补
- 不确定是否值得标记？先标，周日review时再决定

**标记类型：**
- `#insight` - 对自己、生活、工作的感悟
- `#idea/content` - 可能写成内容的想法
- `#idea/product` - 产品/项目想法
- `#seed/story` - 可以讲的故事
- `#seed/observation` - 有趣的观察
- `#fleeting - 需要进一步思考的一些闪念

### 3. 处理流程

```
Daily Journal (标记)
    ↓ 每周日
Weekly Review (提取)
    ↓
Insights Pool / Ideas Backlog / Content Seeds（自动生成）
    ↓ 每周选择 行动 / 创作
```
